# AngeMedia Gateway v0.1.0 Release Notes

This is the first branded AngeMedia release.

## Brand

- Project name: **AngeMedia Gateway**
- Product UI: **AngeMedia Studio**
- Suggested repository name: `angemedia-gateway`

The previous experimental project name was Image Proxy Gateway.

## Core features

- OpenAI-compatible image generation endpoint
- Agnes video generation endpoint
- Default image fallback chain
- Optional Agnes image and OpenAI-compatible image providers
- Automatic remote media localization into `/generated/`
- Lightweight route API
- Lightweight prompt enhancement API
- Agent Skill docs split into image, video, routing, and prompt enhancement documents
- Built-in Web Studio

## Release readiness updates

- Added a grouped Chinese Admin configuration center.
- Polished Studio and Admin UI with a lighter system-settings layout, preview-first result area, and clearer Chinese configuration grouping.
- Added browser-side "download to local" actions for generated images/videos, generated-file management, upload records, history entries, and video tasks.
- Studio now keeps raw JSON collapsed by default, shows a breathing pixel-assembly generation animation, and fades completed media into the preview area.
- Studio now includes a visible browser-side generation queue; duplicate clicks on the same job are blocked, and additional jobs are queued instead of submitted concurrently.
- Ange assistant now presents concrete plan details, prompt changes, work steps, and an inline "confirm and execute" action when pre-generation confirmation is enabled.
- Admin provider management now supports multiple custom OpenAI-compatible image channels with templates, sorting, test, enable/disable, and delete actions. Built-in providers can be moved out of the active model list and restored without deleting secrets.
- Admin assistant settings can pull OpenAI-compatible model lists and test LLM connectivity before saving a model choice.
- History, video queue, and generated-file management now use preview-first media cards with provider, model, request model, and duration metadata.
- Queue completion is now tied to an accessible media preview; asynchronous video submissions stay in submitted/queued state until a real video URL is available.
- Assistant LLM planning accepts fenced JSON output from OpenAI-compatible/Ollama backends and returns clear Chinese errors for non-JSON upstream failures.
- Added `/v1/admin/config-metadata` for front-end configuration labels and field descriptions.
- Added `/v1/admin/provider-templates`, `/v1/admin/providers/{id}/enabled`, `/v1/admin/providers/{id}/sort`, `/v1/admin/providers/{id}/test`, `/v1/admin/assistant/models`, and `/v1/admin/assistant/test`.
- Split FastAPI routes into page, admin, media, and storage modules while keeping existing API paths compatible.
- Added stricter admin config validation for booleans, numbers, and HTTP(S) URLs.
- Split Admin front-end code into API, configuration rendering, and main interaction modules.
- Clean release packaging excludes Python caches, virtual environments, local env files, and test caches.
